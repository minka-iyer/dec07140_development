WCAG Success Criteria Summaries

Each item includes a plain-language aim and specific actions most relevant to our context.

Principle: Perceivable

1.1 Alternative text

Aim: Ensure non-text content has text alternatives for screen reader users.

  • Provide concise, meaningful alt text for images and graphics such as icons, flowcharts, infographics, logos, and buttons.
  • Use empty alt code for decorative images.
  • Label form icons/controls with text descriptions.

1.3.2 Meaningful Sequence

Aim: The order of content can be understood by more people.

  • Use h1 and h3 headings in order so content flows logically.
  • Pair related elements, such as label with input so forms read in the right sequence.

1.4.1 Avoid Colour-Only Cues

Aim: Don’t rely on colour alone to convey meaning.

  • Add text, icons, or patterns in addition to colour, especially for non-native English speakers.
  • Use a contrast ratio of 3:1 between text and background and provide additional visual cues for hover features.
  • Ensure that information conveyed by colour differences is also available through other means (e.g., text labels).

1.4.3 Perceivable Text Contrast

Aim: Text should meet minimum contrast thresholds for readability for more users.

  • The visual presentation of text and images of text has a contrast ratio of at least 4.5:1
  • Use high-contrast colors for essential text.

1.4.4 Resize Text

Aim: Content remains clearly visible and functional at 200% zoom.

  • Use responsive units (e.g., em, rem, %) for text sizing.
  • Ensure no content loss of function occurs at 200% zoom.

Principle: Operable

2.1.1 Keyboard

Aim: All functionality is keyboard accessible.

  • Ensure focusable controls that can be viewed by 'tabbing'.
  • Provide visible focus and logical tab order.

2.4.7 Focus Visible

Aim: Keyboard focus is clearly visible.

  • Custom focus styles with sufficient contrast and size.
  • Never remove outlines without providing a clear alternative.

Principle: Understandable

3.1.5 Reading Level

Aim: Provide supplemental support for complex content.

  • Use plain language; define jargon and acronyms.
  • Offer summaries or glossaries for advanced content.

3.2.3 Consistent Navigation

Aim: Navigation appears in a consistent order and location.

  • Keep menus, headers, and footers consistent across pages.
  • Avoid unexpected reordering between pages.

3.2.4 Consistent Identification

Aim: Identical components are identified consistently.

  • Label similar controls the same way across pages.
  • Keep icon meanings and behaviors consistent.

Relevance and Rationale

The following maps each selected Success Criteria to specific Client/User Needs and gives a brief rationale.

WCAG 2.0 - Criteria Applied to This Site
Criteria User need Rationale
1.1.1 Alternative Text (Non-text Content) Chan and Ivan - Screen-reader compatibility; Miryam - images/diagrams in transcripts. Provide meaningful alt text (or aria-label) so non-visual users get the same information as sighted users.
1.3.2 Meaningful Sequence All users - Ease of accessing interview content in the right order, especially for users that require ideas presented in a logical order. Ensure DOM order matches visual order (headings, question - response) so screen readers read it logically.
1.4.1 Avoid Color-only Cues Chan - doesn’t miss cues when learning. Pair colored highlights with text/labels/icons so meaning isn’t conveyed by color alone.
1.4.3 Perceivable Text Contrast All - Comfortable reading; Chan - reduces fatigue; Ivan - predictable legibility. Meet 4.5:1 contrast for text (e.g., blue tooltip with white text) to keep content readable.
1.4.4 Resize Text (without loss) Chan - can zoom content; Miryam - long passages; Ivan - prefers larger, steady layouts. Use relative units (rem/em) and fluid layouts so zooming to 200% doesn’t clip or overlap content.
2.1.1 Keyboard Navigability Ivan - predictable, structured navigation; All - non-mouse operation. All links, TOC, and interactive highlights are operable via keyboard (Tab/Shift+Tab/Enter/Escape).
2.4.7 Focus Visible Ivan - doesn’t get lost while tabbing; Chan - clear indicator of where she is. Provide a visible focus style (outline/underline) on links and interactive elements, including tooltips.
3.1.5 Reading Level Chan - reduced jargon; Miryam - summaries for dense content. Use plain language in UI; add short explanations for technical terms in tooltips/footnotes.
3.2.3 Consistent Navigation All - quick orientation across pages/sections. Keep TOC placement/labels consistent; include “Back to top” in each interview section.
3.2.4 Consistent Identification All - recognize repeating components quickly. Use the same styling/labeling for repeated elements (e.g., “Question”, “Response”, tooltip highlights).
