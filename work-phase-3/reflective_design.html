Accessibility Audit

This accessibility assessment was conducted using the IBM Equal Access Accessibility Checker Chrome extension. Below are three key issues identified and the actions planned to address them in the final implementation.

Issue 1: Missing Landmark Elements

  • WCAG Criterion: 1.3.1 Info and Relationships (Level A)
  • Affected Element: Main content not wrapped in a landmark element
  • Impact: Screen-reader users cannot easily navigate between sections of the page.
  • Planned Fix: Wrap primary page content inside semantic landmarks such as main,header, and footer; to define structure clearly.

Issue 2: Headings Outside of Landmarks

  • WCAG Criterion: 2.4.6 Headings and Labels (Level AA)
  • Affected Element: <h1> heading located outside of any <main> or <section>
  • Impact: Disrupts logical document flow, causing confusion for users navigating via screen readers.
  • Planned Fix: Move all headings inside relevant landmark regions (e.g., within <main>) and maintain a consistent heading hierarchy.

Issue 3: Table Headers Not Properly Defined

  • WCAG Criterion: 1.3.1 Info and Relationships (Level A)
  • Affected Element: Complex table missing <th> and scope attributes
  • Impact: Screen readers cannot associate table headers with their corresponding cells, making the table content inaccessible.
  • Planned Fix: Add appropriate <th scope="col"> and <th scope="row"> tags to define relationships between headers and cells.

Addressing these issues will improve the site‚Äôs navigation, readability, and accessibility for users with assistive technologies. In the final implementation, I will continue validating each page using the IBM Equal Access Checker to ensure compliance with WCAG 2.2 standards.
